01 / Foundations

Design Tokens

Every color, size, and spacing value is a token. Tokens use HSL values so the entire theme shifts by changing 3 base hue values. Players control --primary-hue, --secondary-hue, and --accent-hue — everything derives automatically. Semantic colors are never themed.

Token Architecture

The theming system works through HSL hue rotation. When a player changes their primary color, only --primary-hue changes. All derived values (dim, glow, ghost, surface, backgrounds, text) recalculate automatically. Semantic colors (--success, --warning, --danger, --info) are constants — they bypass the theme system entirely.

09 / HDRP

HDRP Effects Guide

Unity HDRP gives access to bloom, emissive materials, and HDR color values for UI. Here's how to use them tastefully.

Bloom on UI

Set glowing UI elements to HDR values (intensity > 1.0). Borders: 1.5x. Active tabs: 2.0x. Critical warnings: 2.5x. Body text stays at 1.0x. Bloom threshold: ~0.8.

Emissive Materials

Panel borders, accent bars, and side glow strip use emissive materials. Intensity: borders 1.2, accent bars 2.0, active states 2.5, hover 1.8. Holographic top beam: subtle pulse (3s, 0.8–1.0 range).

HDR Color Values

For glowing elements, multiply color by intensity: Color(0, 0.9, 1.0) * 2.0. Apply HDR to borders, button hovers, health bars, badges. Never apply HDR to backgrounds or large fills — only edges, lines, and small accents.

Glow Intensity Scale (Unity)

Subtle: 1.2 — Idle borders. Medium: 1.8 — Hover, health bars, active tabs. Strong: 2.5 — Selected items, primary actions. Intense: 3.5 — Critical warnings, damage alerts. Maps to CSS box-shadow: 8px → 16px → 30px → 50px.

Panel Shadows

Drop shadow: rgba(0,0,0,0.5), 8px blur, 32px spread. Elevated (modals): 0.7 opacity, 40px spread. Holographic panels: additional drop-shadow filter for ghostly glow.

10 / Motion

Animation Guidelines

Panel Open

Main panels: Fade + slide up 8px, 350ms ease-out. Stagger children 30ms. Modals: Scale 96%→100% + fade, 300ms. Flash border glow. HUD: Slide from screen edge, 250ms. Data panels: Simple fade, 200ms.

Panel Close

Reverse of open but 30% faster. Fade out + slide down 4px. Modals scale to 98% + fade. Close should feel snappy.

Hover & Interaction

Buttons: Border brightens + glow, 200ms. No scale. Data rows: Background tint, 120ms. Tabs: Color shift 200ms. Active indicator slides. List items: 2px left-shift + highlight, 150ms.

Ambient

Glow pulse: Opacity 0.6→1.0, 3s cycle on active borders. Scanlines: Optional 0.5px/s vertical scroll. Health bars: Smooth width 350ms. Critical (<20%): slow red pulse.

Timing

Fast (hover): 120ms. Normal (tab): 200ms. Panels: 300-350ms. Open easing: cubic-bezier(0.16, 1, 0.3, 1). Symmetric: cubic-bezier(0.65, 0, 0.35, 1).

12 / Personalization

Theming System

Players control the UI through three hue sliders. HSL tokens ensure contrast ratios hold at any hue.

187°
220°
35°
Unity Implementation

Store primaryHue, secondaryHue, accentHue (0–360) in PlayerPrefs. ThemeManager singleton recalculates via HSL→RGB. Materials reference shared _ThemeColor properties. Use MaterialPropertyBlock for per-element overrides. Semantic colors are constants bypassing the theme.

Contrast Safety

Background lightness: fixed 4%–17%. Text lightness: fixed 32%–95%. Only hue rotates. Guarantees 4.5:1 body text contrast, 3:1 for display text at any hue. Boost saturation slightly for yellow hues (50°–70°) which can appear washed.

Preset Themes

Default Cyan (187°, 220°, 35°) · Imperial Red (350°, 240°, 35°) · Verdant (140°, 180°, 45°) · Solar Gold (45°, 200°, 15°) · Phantom Purple (270°, 220°, 320°) · Arctic (200°, 210°, 190°)

13 / Rules

Design Guidelines

Do

Use tokens for every color — never hardcode hex (except semantics). Keep panels opaque. Orbitron for titles/buttons only. Share Tech Mono for all numbers. Primary action right, secondary left. Use the correct panel style per context. Test with 3+ theme hues. Min click target 32px. Semantic colors for intended meaning only.

Don't

More than one btn-filled per panel. HDR bloom on large surfaces. Mix panel styles within one panel. --primary for errors. Scanlines on data-heavy panels. Animate content being read. More than 3 font weights per panel. Glow competing with text. Theme semantic colors.

Panel Sizing

Small (tooltips, cards): 200–320px, fixed. Medium (info panels): 360–500px, draggable. Large (fleet, social): 600–900px, resizable, min 400×300px. Full-page (build, tech, VIX): fill screen with 40–60px margin, close button top-right.

Flexalon Tips

Use flexible layout for data rows, lists, grids. Set min sizes on text containers. Tab bar outside scroll area. Map Flexalon padding/margin to spacing tokens. Sidebar: fixed width, content: flexible fill.

Accessibility

Option to disable scanlines and glow pulses. Support UI scaling (80%, 100%, 120%, 150%). 4.5:1 contrast on body text (guaranteed by tokens). Don't rely only on color — pair with text/icons. Keyboard-navigable for future controller support.
