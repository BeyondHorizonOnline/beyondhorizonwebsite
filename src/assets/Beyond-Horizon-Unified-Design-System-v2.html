01

Design Tokens

Every value is a token. HSL-based architecture allows the entire theme to shift via 3 hue values. Players control --ph (primary), --sh (secondary), --ah (accent). Everything derives automatically. Semantic colors are fixed constants.

Token Architecture

Primary hue rotation drives: accent color, borders, glows, panel tints, text accents, button colors, scrollbar styling, and all interactive states. Background colors use a fixed dark navy base (hue 218–220°) that doesn't rotate — this ensures readability at any theme hue. Text lightness values are locked (32%–95%) guaranteeing 4.5:1 contrast. Semantic colors bypass the theme entirely.

Splitter Behavior

Splitters follow the cursor exactly during drag (no transition). On hover, the grab indicator pill fades in over 150ms. While dragging, splitter brightens to --primary with glow. Content in both panes reflows live. If a pane hits its minimum size (120px for vertical, 80px for horizontal), the splitter stops — same rubber-band feedback as resize handles (4px over-drag, spring back 150ms). Save splitter positions per panel to player preferences.

Unity Implementation

Use a child RectTransform as the splitter handle. On drag, adjust flexibleWidth / flexibleHeight of sibling LayoutElements, or directly adjust sizeDelta. For Flexalon: use a FlexalonFlexibleLayout with the splitter modifying the basis values of each child. Ensure LayoutRebuilder.ForceRebuildLayoutImmediate is called on the parent during drag for smooth reflow.

Multi-Panel Flows

Some interactions cascade across multiple sub-panels that appear sequentially left-to-right. Each step reveals the next panel with a slide-in animation. The user can go back or cancel at any point, which collapses panels right-to-left.

Flow Pattern: Left → Middle → Right

How It Works

The parent panel contains a flex row of sub-panels. Initially only the leftmost panel (Step 1) is visible. When the user makes a selection, the next panel slides in from the right (300ms, EaseOutQuad). Previous panels stay visible but may narrow. Each panel has a minimum width — if the container can't fit all panels, earlier steps compress to their minimum or show a collapsed "breadcrumb" state. Cancelling or going back collapses the rightmost panel (200ms, EaseInQuad).

Flow Compression Behavior

When Space Runs Out

When the container can't fit all panels at their comfortable width, panels compress in priority order: Step 1 compresses first (list items truncate, secondary text hides). Step 2 compresses next (percentage buttons wrap). Step 3 never compresses below 180px — if it can't fit, the container scrolls horizontally instead. On narrow screens, consider switching to a stacked/modal approach where Steps 2 and 3 become overlapping modals instead of side-by-side panels.

Sell vs Transfer vs Buy Variations

TabStep 1Step 2Step 3Final Action
SellResource list (station inventory)Set quantity + see credit valueNone — confirms directlySell (filled)
TransferResource list (station inventory)Set quantitySelect destination shipTransfer (filled)
BuyResource list (market/available)Set quantity + see credit costNone — confirms directlyBuy (filled)
Key Difference

Sell and Buy are 2-step flows (no ship selection). Transfer is a 3-step flow. The "action" button in Step 2 changes label: Sell shows "Sell for X credits" (filled, success color), Buy shows "Buy for X credits" (filled, warning color), Transfer shows "Select Ship →" (outlined primary, opens Step 3). This keeps the user oriented — the button always tells them what the next action is.

Unity Implementation

Architecture

Use a parent HorizontalLayoutGroup (or Flexalon row). Each step panel is a child with a LayoutElement. To show: tween preferredWidth from 0 → target over 300ms + fade CanvasGroup alpha 0→1. To hide: reverse. Stagger close animations by 50ms when cascading. Keep selection state in a FlowController that tracks current step, selected resource, quantity, and target ship. On tab change, reset all state and collapse panels 2+3.

// Multi-panel flow controller pseudocode public class StationFlowController : MonoBehaviour { [SerializeField] FlowPanel[] steps; // Step1, Step2, Step3 public void ShowStep(int index) { // Close any panels to the right first for (int i = steps.Length-1; i > index; i--) steps[i].CollapseWithDelay((steps.Length-1-i) * 0.05f); // Then open the target steps[index].Expand(0.3f, Ease.OutQuad); } public void CloseFrom(int index) { for (int i = steps.Length-1; i >= index; i--) steps[i].CollapseWithDelay((steps.Length-1-i) * 0.05f); } }

HDRP Effects

Unity HDRP bloom, emissive materials, and HDR color values for UI elements. Tasteful restraint — glow supports hierarchy, never overwhelms.

Bloom Settings

Bloom threshold: ~0.8. Catches only intentional bright elements. Set UI elements to HDR values (intensity > 1.0). Borders: 1.2x. Active tabs: 2.0x. Critical warnings: 2.5x. Body text stays at 1.0x.

Emissive Materials

Panel borders, accent bars, side glow strips use emissive materials. Intensity: borders 1.2, accent bars 2.0, active states 2.5, hover 1.8. Holographic top beam: subtle pulse (3s, 0.8–1.0 range).

HDR Color Scale

Subtle: 1.2 — Idle borders, dividers. Medium: 1.8 — Hover, health bars, active tabs. Strong: 2.5 — Selected items, primary buttons. Intense: 3.5 — Critical warnings, damage alerts. In Unity: Color(0, 1.0, 1.0) * intensity. Maps to CSS shadow: 8px → 16px → 30px → 50px blur.

Panel Shadows

Standard: rgba(0,0,0,0.5), 8px blur, 32px spread. Elevated (modals): 0.7 opacity, 40px spread. Holographic: additional drop-shadow filter for ghost glow effect.

Never Do

Apply HDR/bloom to backgrounds, large surfaces, or body text. Only edges, lines, accent bars, small indicators, and interactive highlights. Large emissive areas wash out the entire screen.

Unity Implementation

Store primaryHue, secondaryHue, accentHue (0–360 float) in PlayerPrefs. ThemeManager singleton recalculates all derived colors via HSL→RGB. Materials reference shared _ThemeColor properties. Use MaterialPropertyBlock for per-element overrides. Boost saturation slightly for yellow hues (50°–70°) to prevent washout.

Contrast Safety

Background lightness: fixed 4–18% (dark navy). Text lightness: fixed 32–95%. Only hue rotates. Guarantees 4.5:1 for body text, 3:1 for display text at any hue. Test with Imperial Red (350°) and Solar Gold (45°) — these are the hardest to read.

Rules & Checklist

Follow when building any new UI. Violations break theme switching and visual consistency.

✓ Always

Image.color = WHITE on PanelGlow materials. Use ThemeableText for standalone text. Reference UIThemeData — never hardcode hex. Test all preset themes. Orbitron for headers/buttons, Rajdhani for body/tabs, Share Tech Mono for data. Opaque panel backgrounds. Primary action right, secondary left. Min click target 32px. Save resizable panel positions. Pin headers/inputs so always visible. Use correct panel style per context. Use SciFiButton shader + MAT_SciFiButton_Filled for all buttons.

✗ Never

ThemeableText inside ThemeableButton/InputField/ListItem (they self-manage). GlowIntensity = 0 (breaks hierarchy). Orbitron for body text. Custom one-off colors — add to UIThemeData. Skip PanelGlow shader. Multiple filled buttons per panel. HDR bloom on large surfaces. Mix panel styles in one panel. Use --primary for errors. Scanlines on data-heavy panels. Animate content being read. Allow panels fully off-screen. Theme semantic colors.
